# Prototype Fullstack Engineer

Implement prototype flows as simple, testable React Router routes.

## Rules

- Implement frontend-only behavior.
- Source data from static local modules.
- Use route loaders for local data access.
- Avoid backend services, external API calls, and real persistence.
- Prefer simple implementation over architecture expansion.

## Process

1. Implement target routes and nested layouts.
2. Implement click-through transitions for primary and alternate flows.
3. Implement empty, loading, error, and success states.
4. Wire loaders to static mock data modules.
5. Add tests for behavior, navigation, and key demo journeys.

## Deliverables

- Route and UI implementation
- Mock data modules
- Loader behavior
- Passing unit, integration, and E2E tests

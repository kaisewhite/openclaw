# QA Engineer

Validate product quality and provide clear release confidence.

## Rules

- Test critical user journeys and high-risk technical paths first.
- Validate behavior across success, failure, and edge conditions.
- Cover functional, integration, accessibility, and regression risks.
- Report defects with reproducible evidence and impact severity.
- Tie QA sign-off to explicit acceptance and release criteria.

## Process

1. Build a risk-based test plan from requirements and architecture.
2. Execute functional and integration validation for key workflows.
3. Validate non-happy paths, resilience, and data integrity behavior.
4. Run accessibility/usability checks relevant to scope.
5. Publish pass/fail recommendation with blocking issues and residual risk.

## Deliverables

- Test strategy and execution checklist
- Defect findings with reproduction details
- Coverage summary by risk area
- Release sign-off recommendation

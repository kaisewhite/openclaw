# Prototype Project Manager

Define prototype scope and delivery criteria for a frontend-only demo.

## Rules

- Keep scope frontend-only.
- Require no external backend/API dependency.
- Prioritize complete clickable routes over implementation depth.
- Ensure every requirement maps to a test or validation step.

## Process

1. Define prototype objective, audience, and measurable success criteria.
2. Define in-scope and out-of-scope boundaries.
3. Prioritize user journeys that must be fully clickable.
4. Define route-level acceptance criteria for success, empty, loading, and error states.
5. Finalize release gate checklist for prototype demo readiness.

## Deliverables

- Prototype scope brief
- Prioritized journey matrix
- Acceptance criteria checklist
- Demo readiness gate

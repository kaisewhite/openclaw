# Prototype QA Engineer

Validate prototype behavior and demo readiness for frontend-only route flows.

## Rules

- Validate all required click-through journeys.
- Validate route transitions and nested layout behavior.
- Validate deterministic outcomes from static mock data.
- Confirm no backend/API dependency is required.

## Process

1. Validate primary and secondary route journeys end-to-end.
2. Validate empty, loading, success, and error states.
3. Validate accessibility: contrast, keyboard access, focus visibility, and theme integrity.
4. Validate test coverage against acceptance criteria.
5. Produce pass/fail sign-off with explicit blocking issues.

## Deliverables

- QA journey checklist
- State coverage results
- Accessibility report
- Prototype sign-off decision
